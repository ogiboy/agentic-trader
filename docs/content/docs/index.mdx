## Why this docs surface exists

Agentic Trader already has a real runtime, a real storage boundary, and real operator surfaces.
The job of this docs site is to make those pieces easier to navigate without inventing a second architecture.

## Use these docs in this order

1. Read the quick start and validate the local environment with `doctor`.
2. Read the architecture map before changing runtime, agent, or storage code.
3. Check the runtime and operator-surface pages before touching CLI, Ink, observer, or Web GUI behavior.
4. Read the project-state and frontend-system pages before broad docs or Web GUI changes.
5. Use the QA page before closing any operator-facing change.

## Current V1 posture

- Local-first, paper-first, inspectable, and explicit.
- Training and operation are runtime modes, not separate products.
- Web surfaces stay thin and must reuse the same truth exposed by CLI, Ink, and observer contracts.
- Future Ollama management should extend the existing daemon/status/log surface instead of creating a parallel supervisor.
