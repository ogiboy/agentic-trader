## Current posture

Agentic Trader is already a real local-first runtime with:

- a staged specialist graph
- shared runtime and review contracts
- DuckDB-backed persistence
- CLI, Rich, Ink, observer, and Web GUI surfaces
- paper-first execution discipline

This docs site should help contributors find those boundaries faster.
It should not become a second architecture that drifts from the repository.

## Current hardening track

The near-term V1 work is centered on:

- hardening the existing runtime and operator surfaces instead of rewriting them
- making model-service and provider health more visible
- standardizing onboarding and bootstrap flows
- improving developer docs, QA evidence, and frontend consistency

## What is open right now

- optional app-managed Ollama supervision through the same daemon and log surfaces already used by runtime status
- a provider-aware cross-platform bootstrap flow for macOS, Linux, and Windows
- incremental Web GUI migration from legacy shell classes toward shadcn primitives and Tailwind tokens
- more developer docs around contributing, reference flows, and project-state snapshots

## Guardrails worth repeating

- paper remains the default backend
- runtime failures should stay visible
- fallback paths must not fake confidence or execution safety
- web surfaces stay thin and must reuse the same runtime truth as CLI, Ink, and observer
