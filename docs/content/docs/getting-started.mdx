## Baseline environment

The current repository still expects a Python plus Node workflow.
Today the canonical setup is still the repository quick start, while a smoother provider-aware bootstrap flow is planned for V1 hardening.

```bash
conda create -n trader python=3.14 poetry
conda activate trader
poetry env use "$CONDA_PREFIX/bin/python"
poetry install --extras dev
```

## Validate the environment first

Run the smallest truth check before opening UI surfaces:

```bash
python main.py doctor
```

Then validate the installed command if you rely on it:

```bash
agentic-trader doctor
```

## Configure local model access

The project is still Ollama-first by default.
The repository-level runtime settings follow the tracked-example plus local-override pattern, so copy `.env.example` to `.env.local` before filling real secrets or machine-specific values.
If you want the Fumadocs feedback widget to forward into GitHub Discussions, copy `docs/.env.example` to `docs/.env.local` and add the GitHub App credentials there.
Typical environment variables:

```bash
export AGENTIC_TRADER_MODEL=qwen3:8b
export AGENTIC_TRADER_BASE_URL=http://localhost:11434/v1
```

Do not assume Ollama traffic alone means the runtime is healthy.
Model-service reachability, runtime status, and recent logs should all agree.

## First useful commands

Run one strict cycle:

```bash
python main.py run --symbol AAPL --interval 1d --lookback 180d
```

Open the primary Ink control room:

```bash
agentic-trader
```

Open the Web GUI:

```bash
cd webgui
npm install
npm run dev
```
